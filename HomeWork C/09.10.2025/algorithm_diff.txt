1. сложность алгоритма recursive_solution. Первый ферзь может быть расставлен n способами, последущие соответственно n-1, n-2 ..... 1 способом, => сложность алгоритма n!
2. сложность алгоритма iterative_solution. Для проверки расположения ферзей используется 3 двойных вложенных цикла => сложность алгоритма - n^2
3. сложность алгоритма bit_solution. Тоже рекурсивное решение, сложность n!
